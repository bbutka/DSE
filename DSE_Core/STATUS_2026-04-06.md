# DSE_Core Status

Date: April 7, 2026

## Current integrated flow

The active integrated package flow is now:

1. Phase 1: math optimisation by default via `dse_tool.agents.phase1_mathopt_agent.Phase1MathOptAgent` with CP-SAT selected by default
2. Phase 1 fallback: ASP/Clingo via `dse_tool.agents.phase1_agent.Phase1Agent`
3. Phase 2: Clingo via `Clingo/zta_policy_enc.lp`
4. Phase 3: Clingo via `Clingo/resilience_enc.lp`
5. Runtime: optional tc9-specific extension after or alongside Phase 2

## Remaining limitations

- Runtime remains tc9-specific by design.
- Legacy compatibility tolerance still exists for:
  - `ungovernerd_pep` parser support
  - `max_bufg` alias support
- End-of-suite CPU usage can still drop once the last long solver case becomes the only remaining active worker. The parallel runner reduces this effect by front-loading heavy tests, but it cannot eliminate the final long-tail case without further splitting that test.
